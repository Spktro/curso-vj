Frames por segundo

60 FPS = 60 veces por segundo

Godot corre por defecto a 60 FPS. Todo el código dentro del game loop se ejecuta 60 veces por segundo.

Arranque del nodo

_ready(): el momento de preparación

Godot la llama una sola vez, justo cuando el nodo entra a la escena. Sirve para configurar todo antes de que empiece la acción.

extends Node2D

var vida = 100
var nombre = "Héroe"

func _ready():
    position = Vector2(100, 300)
    print("=== Juego iniciado ===")
    print("Vida: " + str(vida))
_ready()

Regla de oro

🎤 _ready() es como el discurso de apertura: se dice una vez, al principio, y después el juego toma su curso.

👐 Mini práctica

Cambiá la posición inicial y los valores de las variables, ejecutá el proyecto y observá cómo cambia el punto de partida. La idea: entender que _ready() define el estado inicial del mundo.

El corazón

_process(delta): cada frame

Godot la llama en cada frame. A 60 FPS, se ejecuta 60 veces por segundo. Es el corazón del juego en movimiento.

func _process(delta):
    print("Estoy corriendo constantemente")

Esto llena la consola sin parar — el motor la llama una y otra vez.

La diferencia clave

_ready() vs _process()

FunciónCuándoPara qué
_ready()Una vez, al entrar a la escenaInicialización, posición y valores de partida
_process(delta)Cada frame (~60/seg)Lógica continua: movimiento, input en tiempo real
Acá muchos hacen “click” con el game loop: uno se imprime una vez, el otro sin parar.
El concepto clave

El problema de mover sin delta

func _process(delta):
    position.x += 5

Suma 5 píxeles… ¿por frame?

MáquinaResultado
60 FPS5 × 60 = 300 px/seg
30 FPS5 × 30 = 150 px/seg
⚠️ El mismo juego, en distintas máquinas, se comporta distinto. Problema serio.
delta

La solución: multiplicar por delta

delta es el tiempo (en segundos) desde el frame anterior. A 60 FPS ≈ 0.016.

60 FPS200 × 0.016 ≈ 200 px/seg
30 FPS200 × 0.033 ≈ 200 px/seg
 
func _process(delta):
    position.x += 200 * delta

El movimiento queda atado al tiempo real, no a la velocidad de la máquina.

delta · regla

La regla, sin excepción

📏 Cada vez que escriban movimiento dentro de _process(), multiplican por delta. Siempre.

🏠 Analogía

Es como pagar el alquiler por día en vez de una cuota fija. Tenga el mes 28 o 31 días, siempre terminás pagando lo mismo al final.

Primera app real

Un objeto que se mueve solo

extends Node2D
var velocidad = 150

func _process(delta):
    position.x += velocidad * delta   # → Derecha
    # position.x -= velocidad * delta  # ← Izquierda
    # position.y += velocidad * delta  # ↓ Abajo
    # position.y -= velocidad * delta  # ↑ Arriba
  1. Ejecutar con movimiento a la derecha
  2. Cambiar velocidad 100 → 300
  3. Probar hacia abajo
  4. Combinar dos direcciones (diagonal)
Movimiento

Dirección con Vector2

extends Node2D
var velocidad = 200

func _process(delta):
    # Vector2(x, y): x horizontal, y vertical
    position += Vector2(velocidad, 0) * delta
    # Vector2(-velocidad, 0) → izquierda
    # Vector2(0, velocidad)  → abajo
Por ahora solo mostralo: Vector2 junta las dos coordenadas en un valor. La clase de física lo va a retomar en profundidad.

Segunda capa

Input + process = control real

En la Clase 2 el input solo hacía print(). Ahora se conecta con movimiento real.

El objeto se mueve solo mientras el jugador mantiene la tecla. Eso es control en tiempo real.
func _process(delta):
    if Input.is_action_pressed("ui_right"):
        position.x += velocidad * delta
Input

Las tres variantes — cuándo usar cada una

MétodoDetectaIdeal para
is_action_pressedMientras esté presionadaMovimiento continuo
is_action_just_pressedEl instante en que se presionaSaltar, disparar
is_action_just_releasedEl instante en que se sueltaCargar poderes
Las tres se consultan dentro de _process(), cada frame.
Profesionalizar

Input Map: acciones con nombre

❌ Sin Input Map

if Input.is_key_pressed(KEY_D) repetido en 50 lugares. Cambiar la tecla = cambiar 50 líneas.

✅ Con Input Map

Detectás acciones con nombre (mover_derecha). Cambiás la tecla en un solo lugar y todo sigue funcionando.

Se accede en Project → Project Settings → Input Map.
Input Map

Crear una acción nueva

  1. Escribir el nombre: mover_derecha
  2. Clic en Add
  3. Clic en el + de la acción
  4. Presionar la tecla (ej. D)
  5. Confirmar
if Input.is_action_pressed("mover_derecha"):
    position.x += velocidad * delta

👐 Práctica: creá mover_derecha y mover_izquierda, asignalas a teclas y usalas. Rompé la dependencia de las acciones predeterminadas.

Teclado

Movimiento completo (4 direcciones)

func _process(delta):
    if Input.is_action_pressed("mover_derecha"):
        position.x += velocidad * delta
    if Input.is_action_pressed("mover_izquierda"):
        position.x -= velocidad * delta
    if Input.is_action_pressed("mover_arriba"):
        position.y -= velocidad * delta
    if Input.is_action_pressed("mover_abajo"):
        position.y += velocidad * delta
⚠️ En Godot el eje Y está invertido: Y positivo es hacia abajo. Fuente frecuente de confusión.
Mouse

El mouse como fuente de input

func _process(delta):
    if Input.is_mouse_button_pressed(MOUSE_BUTTON_LEFT):
        print("¡Click izquierdo!")

    var pos = get_viewport().get_mouse_position()
    print("Mouse en: " + str(pos))

Bonus — ir hacia el mouse

if Input.is_mouse_button_pressed(MOUSE_BUTTON_LEFT):
    var objetivo = get_viewport() \
        .get_mouse_position()
    position = position.move_toward(
        objetivo, velocidad * delta)
